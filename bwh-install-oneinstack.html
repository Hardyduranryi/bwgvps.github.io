第一、Oneinstack支持环境以及可安装的系统

提到一键安装软件包，我们可能会说到LNMP系统，这个是发展最早而且一直也在更新的。后面我们也会有一起介绍到。今天先介绍oneinstack一键包，这个发展虽然只有2-3年，但是功能特别强大，一旦有软件更新，官方都会添加到脚本中，而且可以支配的脚本配置很多。比如WEB环境支持lnmp（Linux + Nginx+ MySQL/MongoDB+ PHP）、lamp（Linux + Apache+ MySQL/MongoDB+ PHP）、lnmpa（Linux + Nginx+ MySQL/MongoDB+ PHP+ Apache）、lnmt（Linux + Nginx+ MySQL/MongoDB+ Tomcat）、lnpp（Linux + Nginx+ PostgreSQL+ PHP）、lapp（Linux + Apache+ PostgreSQL+ PHP）、lnmh（Linux + Nginx+ MySQL+ HHVM）。

这个相对我们熟悉的LNMP脚本丰富很多，适合很多不同WEB系统用户的需要安装。但是，一般我们用户只会安装LNMP或者LAMP居多。Oneinstack一键脚本支持在CentOS 6~7、Debian 7～9、Ubuntu 12～18系统中运行。

第二、Oneinstack 一键包安装部署WEB环境过程

Oneinstack部署起来还是很简单的，如果我们有玩过LNMP WEB脚本，应该不是什么难事。全部是可视化的脚本向导过程，虽然没有那种面板这么漂亮，但是基本上还是能看懂的。我们一起试试吧。

1、CentOS/Redhat系统安装方法

yum -y install wget screen

wget http://mirrors.linuxeye.com/oneinstack-full.tar.gz

tar xzf oneinstack-full.tar.gz

cd oneinstack

screen -S oneinstack

./install.sh

2、Debian/Ubuntu系统安装方法

apt-get -y install wget screen

wget http://mirrors.linuxeye.com/oneinstack-full.tar.gz

tar xzf oneinstack-full.tar.gz

cd oneinstack

screen -S oneinstack

./install.sh

我们根据自己安装的系统然后选择对应的脚本，其实也就是yum与apt的区别，其他都是一样的。

这里我们可以看到根据向导都有标注，重置端口、设置安全防火墙，以及选择引擎。可以是NGINX，一般我们都安装这个。

这里我们可以选择数据库，根据需要选择MYSQL数据库，还是MariaDB，以及各种版本支持需要根据实际的项目要求。

这里我们可以选择PHP版本，最高可以是PHP7.3，这里需要说的是，如果我们程序是WORDPRESS，建议直接最高版本，因为在PHP7.0+版本的时候性能最好。同时，我们可以开启缓存机制。

最后我们可以根据需要再安装需要的组件扩展，以及数据库PHPMYADMIN数据库管理客户端。最后回车就可以自动安装。

第三、Oneinstack 安装站点建站过程

我们在上面自动安装部署完毕之后，需要最后重启一下。必须重启才可以生效。完毕之后我们就可以建站，建站使用的是ONEINSTACK脚本自带的工具，我们可以用下面介绍的方法。

1、添加站点

~/oneinstack/vhost.sh

然后我们根据提示进行建站。

完毕之后，我们就可以用SFTP登录服务器，然后添加站点程序中的程序，以及解析完毕之后就可以建站向导安装网站。
